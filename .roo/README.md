This directory contains custom rules and workflows specifically designed for the Django learning web application project.

- `django-development.md` - Standards for Django development including models, views, forms, and templates
- `clean-code.md` - General clean code principles (existing)

- `nano-banana.md` - Image generation guidelines (existing)
- `research.md` - Research and insight gathering standards (existing)

## Usage

These rules and workflows will be automatically applied based on the file types and contexts encountered during development. When working on Django components, the `django-development.md` rule will be applied to ensure consistency and best practices.

For new feature development, use the `django-feature-implementation.md` workflow to ensure all phases are properly addressed.

For model creation, use the `django-model-creation.md` workflow to maintain consistency in your data models.

For development following TDD principles, use the `django-tdd-cycle.md` workflow.
